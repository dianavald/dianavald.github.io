Meteora

Meteora is a collaborative game that engages two people in using their lung capacity to create particles, represented by small LED orbs.

The idea is to control and synchronise these light droplets from two sides to form a perfect balance in the centre. When theyâ€™re in harmony, the users are able to initiate a series of collisions where the particles explode into an elaborate display of colours.

The installation focuses on transforming air streams into energy by means of a collaboration between two people. And at the heart of this artwork lies the concept of universal balance between things in motion. The aim is to provoke interaction, participation and dialogue.

The installation is contained within a wooden box with a plexiglass display. The LED strips are placed behind the glass. The input uses a fan contained within a 3D printed shape. The 3D printed shape facilitates better flow of air to the fan. The fan acts as a sensor and converts the air blown by the user into electricity which then lights up the LED on the panel.

The main purpose of this project was to create a game for a total of two players and 1 minute of gameplay. We have been asked to create a light based game using LEDS and sensors, and we started to think about the gaming interface: the controller. Our main concerns were about the durability and the stress resistance of the product, so we started searching for a touchless nterface. We also wanted to remain coherent to the concept of "making".

So, instead to use a "one to one" interface (a microphone or a joystick, as an instance), we decided to twist a device and turn into something else. And we made two fans become generators, using the generated voltage as a controller.

For the programming side, we created a particles based system, with a particles manager who takes care of every particle behaviours. Written in C++ / C language, the code has been optimised for the use on Arduino boards, which unlike nowadays computers don't have a massive computational power. We started developing with a Uno board, and when the required calculation (even with the strong optimisation) was overloading the board's 2kB RAM memory, we switched on a more powerful Arduino MKRZero, which packs 32kB of RAM.

Unlike the Arduino Uno board, the MKRZero only handles 3.3V of input voltage, and for this we had to implement a voltage divider circuit for each fan output, in order to prevent any excessive voltage on the pins.

This project was developed for the Arduino booth at the Maker Faire, Rome 2017.
